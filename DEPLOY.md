# 国内静态托管部署说明

本项目是纯前端静态网页，不需要服务器运行时，也不需要构建步骤。

## 推荐方案：腾讯 EdgeOne Pages

适合当前这个轻量网页：直接托管静态目录，移动端用链接访问。

### 方式一：连接代码仓库

1. 将当前工程推送到一个 Git 仓库。
2. 在 EdgeOne Pages 新建项目，选择导入该仓库。
3. 构建设置填写：
   - 框架预设：静态站点或 Other。
   - 构建命令：留空。
   - 安装命令：留空。
   - 输出目录：`web/mandala`
4. 部署后访问平台分配的默认域名。

### 方式二：直接上传静态文件

如果平台入口支持直接上传静态文件：

1. 上传 `web/mandala` 目录内的全部内容。
2. 确认入口文件是 `index.html`。
3. 部署完成后访问平台分配的链接。

## 备选方案：腾讯云 CloudBase 静态网站托管

适合已经在用腾讯云账号的情况。

1. 开通 CloudBase 环境。
2. 进入静态网站托管。
3. 上传 `web/mandala` 目录内容。
4. 默认首页设置为 `index.html`。

## 移动端访问注意

- 如果绑定自有域名，国内平台通常需要实名认证；自有域名解析到中国大陆节点时通常还涉及 ICP 备案。
- 如果只使用平台分配的默认访问域名，一般可以先快速预览，不必马上购买服务器。
- 当前网页的播放控制支持键盘和触控：移动端双击播放/暂停，右侧上下滑调整时间。
